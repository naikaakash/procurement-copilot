# 🧑‍🚀 Aalok's Sidekick — AI Dev Memory System

Welcome to **Aalok's Sidekick**, a personal app-building memory assistant. This system captures how Aalok builds apps, stores design decisions, reusable prompts, architecture patterns, bug fixes, UI/UX preferences, and phase-end lessons so that any future AI coding tools (such as Gemini, OpenCode, ChatGPT, Claude Code, or Cursor) can inherit the collective experience of "old me."

## 📂 System Map

* **`/aaloks_sidekick`** (Root): System rules, triggers, and usage guides.
* **`/ai_memory`**: Reusable patterns, prompt libraries, journals, bug fixes, and deployment notes.
* **`/dev_agents`**: Standard developer persona prompts and memory/bootstrap agents.
* **`/project_docs`**: Standardized project constraints, feature scopes, data contracts, and changelogs.
* **`/starter_kits`**: Reusable templates for procurement workbenches, SaaS dashboards, and QA checklists.

## 🚀 The Core Philosophy
1. **Visibility & Trust First**: Prioritize tabular data, dynamic filtering, and deep visibility. Chatbots and automation are secondary and come later.
2. **Data & Service Layer Decoupling**: Never hardcode file-parsing or API logic in the frontend. All calculations and KPIs must go through a service layer.
3. **Traceability**: All dashboard counts and counts must be explainable and verifiable. No mock data or hidden assumptions in production.
