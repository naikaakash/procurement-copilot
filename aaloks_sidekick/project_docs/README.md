# 📋 Project Docs Folder

This folder holds project-specific standards, scope maps, and change validations.

* **`PROJECT_RULES.md`**: Strict codebase level engineering rules.
* **`architecture.md`**: Decoupling, routing, and data flow layers.
* **`data_contract.md`**: Typed API definitions.
* **`feature_scope.md`**: Deliverables and out-of-scope markers.
* **`ui_standards.md`**: Dark theme design parameters.
* **`validation_checklist.md`**: Checklists for PM/BA/QA review.
* **`known_issues.md`**: Outstanding glitches or system assumptions.
* **`changelog.md`**: Log of all features shipped.
