## How to Use This Catalog

1. Run the generator: `node scripts/generate-scenario-csv-data.js`
2. Start the app: `npm run dev`
3. Trigger data refresh: POST `/api/control-tower/sync-erp` (or via Control Tower UI)
4. Navigate to the workbench page listed for each scenario
5. Verify the **Expected Result** column

> [!IMPORTANT]
> **Schema invariants** — enforced by both the generator and `azure-csv-sync.ps1`:
> - `po_schedule_lines.csv` must **NOT** contain `confirmation_control_key`
> - `purchase_order_items.csv` **MUST** contain `confirmation_control_key`

## Regenerating the Data

```bash
# Validate only (no file writes)
node scripts/generate-scenario-csv-data.js --validate-only

# Generate CSVs (default seed=42, default out=procurement_data_sample)
node scripts/generate-scenario-csv-data.js

# Custom seed or output folder
node scripts/generate-scenario-csv-data.js --seed 99 --out ./my_test_data
```

## Uploading to Azure (when Azure location is confirmed)

```powershell
# Set credentials (one-time per session)
$env:AZURE_STORAGE_ACCOUNT = "your-account-name"
$env:AZURE_STORAGE_KEY     = "your-key"
$env:AZURE_CSV_CONTAINER   = "procurement-csv"

# Dry run to preview
.\scripts\azure-csv-sync.ps1 -Action Upload -DryRun

# Actual upload (includes pre-upload schema check)
.\scripts\azure-csv-sync.ps1 -Action Upload

# Download Azure CSVs locally (auto-backs up existing local files first)
.\scripts\azure-csv-sync.ps1 -Action Download
```

> [!NOTE]
> The Azure CSV source location is **not yet hardcoded** in the app. The app reads CSVs directly from the local `procurement_data_sample/` folder at runtime via `csvDataService.ts`. The `azure-csv-sync.ps1` script is a **parameterized helper** for when you configure and point an Azure Storage account as the canonical source.
