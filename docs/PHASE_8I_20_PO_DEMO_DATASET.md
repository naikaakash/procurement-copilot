## 🔧 Verification Mocks Sync Flow

You can demonstrate the core **Verification Sync Flow** using **`4500001014`** (Pending SAP Update), **`4500001015`** (Verification Pending), and **`4500001016`** (Resolved):

1.  **Step 1: Recommended Update:**
    *   Open **`4500001014`** in the worklist. Note it asks you to manually shift the delivery date in SAP to `2026-06-15`.
    *   Click **"Confirm manual SAP action completed"**. This moves it to `VERIFICATION_PENDING` (Awaiting Source Refresh).
2.  **Step 2: Sync and Verification:**
    *   To simulate the ERP sync, trigger the `/api/control-tower/sync-erp` endpoint (or call the `/api/recommendations/verify-after-sync` endpoint).
    *   Since **`4500001015`** is awaiting sync and the CSV file still contains the old date `2026-05-20`, its verification will **`FAIL`**.
    *   Since **`4500001016`**'s source CSV file actually contains the new date `2026-06-15`, its verification will **`PASS`**, and the recommendation automatically shifts to **`CONFIRMED_RESOLVED`**.
